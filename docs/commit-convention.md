# 커밋 컨벤션

이 저장소는 작업 과정을 평가 자료처럼 남기는 것이 중요하므로, 커밋을 작게 쪼개서 기록합니다.

## 기본 형식

```text
type : 한국어 설명
```

예시:

```text
docs : 데이터 입력 필드 정의
feat : Trip 단위 feature 생성 로직 추가
fix : 생활권 비율 계산 오류 수정
test : 점수 계산 검증 케이스 추가
```

## 타입

| 타입 | 의미 |
|---|---|
| `docs` | 문서 추가 또는 수정 |
| `feat` | 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 동작은 유지하고 구조 개선 |
| `test` | 테스트 추가 또는 수정 |
| `chore` | 설정, 폴더 구조, 기타 작업 |
| `data` | 샘플 데이터 또는 데이터 스키마 추가 |
| `report` | 결과 리포트 추가 또는 수정 |

## 커밋 분리 기준

- 폴더 구조 추가와 문서 작성은 분리합니다.
- 데이터 스키마 정의와 실제 샘플 데이터 추가는 분리합니다.
- feature 생성, 모델 학습, 점수 계산, 최종 판단 로직은 각각 분리합니다.
- 한 커밋에는 하나의 의도만 담습니다.

## 이번 프로젝트 권장 커밋 순서

```text
chore : 프로젝트 기본 폴더 구조 추가
docs : 프로젝트 개요 작성
docs : 커밋 컨벤션 정리
docs : AI 모델 작업계획 수립
docs : 데이터 입력 필드 정의
data : Trip 샘플 데이터 추가
feat : 생활권 생성 feature 계산 추가
feat : 운전행동 feature 집계 추가
feat : 평소패턴 변화 감지 모델 추가
feat : 최종 점수 계산 추가
feat : 리워드 및 케어 판단 로직 추가
report : 모델 결과 요약 리포트 추가
```
