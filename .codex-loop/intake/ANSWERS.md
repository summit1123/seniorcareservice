# Summit 인테이크 답변

모드: proposal
상태: 확정

## 확정 목표 한 줄

- 생활권 기반 시니어 안심주행 특약의 AI 모델 견본과 제출용 시각화 자료를 source-first 방식으로 만든다.

## 요청자 / 제품

### Q1. 이번 런에서 실제로 만드는 것을 한 문장으로 적으면 무엇입니까?
답변:
- 공공 주행 데이터 구조를 활용해 생활권 생성, 운전행동 feature, 평소패턴 변화 감지, 리워드/케어 판단을 설명하는 모델 견본과 시각화 자료를 만든다.

### Q2. 이 결과물을 가장 먼저 사용할 사람 또는 검토할 사람은 누구입니까?
답변:
- 팀원과 삼성화재 공모전 심사위원.

### Q3. 어떤 조건이 충족되어야 이 첫 버전을 정말 쓸 만하다고 말할 수 있습니까?
답변:
- 데이터 담당자가 필요한 컬럼을 이해할 수 있어야 한다.
- 모델 담당자가 feature table에서 score table과 decision table까지 만들 수 있어야 한다.
- 제안서 담당자가 그림과 표를 바로 발표/리포트에 옮길 수 있어야 한다.

### Q4. 이번 런에서 명시적으로 제외할 범위는 무엇입니까?
답변:
- 실제 보험 요율 산정
- 실제 시니어 개인 데이터 사용
- 네비게이션 실시간 경로 변경 기능
- TAAS 위치기반 링크 API 중심 설계

## 승인 / 검토

### Q5. 이 브리프와 task graph를 최종 승인할 사람은 누구입니까?
답변:
- 동현.

### Q6. COMPLETE라고 말하기 전에 반드시 남아 있어야 할 증거와 산출물은 무엇입니까?
답변:
- Ralph bootstrap 파일
- 데이터 계약서
- AI 모델 작업계획
- 시각화 workflow 문서
- 프로젝트 전용 task graph
- preflight 결과

### Q7. 유혹적이지만 허용할 수 없는 지름길은 무엇입니까?
답변:
- 예쁜 PDF만 만들고 source와 데이터 구조를 비워두는 것
- TAAS를 개인 사고 예측 라벨처럼 과장하는 것
- 생활권 밖 주행을 무조건 위험하다고 단정하는 것
- 고객에게 벌점이나 보험료 인상처럼 보이는 표현을 쓰는 것

## 제출 전략

### Q8. 중요한 입력 항목, 분량 제한, 파일 규칙, 심사 기준은 무엇입니까?
답변:
- 현재 단계에서는 코드 저장소와 팀 공유용 자료를 먼저 정리한다. 제출 형식이 확정되면 proposal source와 PDF render gate에 반영한다.

### Q9. 심사위원에게 어떤 톤과 문체로 써야 합니까?
답변:
- 과장 없이 보험사 직원/심사위원이 검토하기 쉬운 문체. 근거, 표, 흐름, 한계가 명확해야 한다.

### Q10. 반드시 들어가야 할 근거, 레퍼런스, 스크린샷은 무엇입니까?
답변:
- 공공 사업용차량 Trip 데이터 필드와 모델 feature 매핑
- 생활권 생성 AI 구조
- Isolation Forest 기반 평소패턴 변화 감지 구조
- 최종 점수와 의사결정 테이블
- 고객 3명 예시 시각화

## 근거 / 스토리

### Q11. 앞부분에 꼭 밝혀야 할 맥락은 무엇입니까?
답변:
- 기존 마일리지 특약은 주행거리 중심이고, 착한운전 특약은 안전운전점수 중심이다. 제안 특약은 생활권과 평소패턴 안정성을 추가 기준으로 본다.

### Q12. 심사위원이 기억해야 할 구체적 시나리오는 무엇입니까?
답변:
- A고객은 생활권 중심으로 안정 운전해 추가 리워드를 받는다.
- B고객은 생활권 밖 주행이 있어도 위험행동 변화가 없어 기본 유지된다.
- C고객은 최근 생활권 밖 주행과 급감속/과속이 함께 늘어 예방 케어 안내 대상이 된다.

### Q13. 첨부 문서에는 어떤 시각 자료나 증거가 들어가야 합니까?
답변:
- AI 활용 구조도
- 데이터 pipeline diagram
- feature table 예시
- score table 예시
- decision flow
- 고객별 판단 예시 chart
