# 디자인 계약

Preset: document-editorial
Reference-Pack: editorial-signal

## 이 문서의 역할

이 파일은 현재 workspace의 디자인 source of truth입니다.

- 제안서, 제출 문서, PRD, 전략 문서에는 `document-editorial` 을 사용합니다.
- 실제 소프트웨어 UI, 대시보드, 에이전트 제어면에는 `product-ops` 를 사용합니다.
- `Reference-Pack` 에는 Ralph가 참고해야 할 시각 계열을 적습니다.

## 왜 필요한가

명시적인 디자인 계약이 없으면 자율 loop는 쉽게 같은 약한 기본값으로 수렴합니다. 중첩 카드, 장식용 강조색, 약한 위계, 빈약한 정보량, AI가 만든 scaffold 같은 화면이 대표적입니다.

이 문서는 그런 드리프트를 초기에 막기 위해 존재합니다.

## 현재 방향

- 실제 검토자, 구매자, 운영자를 기준으로 씁니다.
- 장식 카드보다 표와 정보 구조를 우선합니다.
- 타이포그래피는 검정 중심, 절제된 분위기, 충분한 밀도를 유지합니다.
- 시각 자료는 증거를 싣거나 구조를 설명할 때만 사용합니다.
- 여전히 autogenerated처럼 느껴진다면 렌더보다 source와 규칙을 먼저 고칩니다.
- 보조 레퍼런스는 `analyst-workbench`입니다. 모델 feature, score, decision table은 분석 워크벤치처럼 비교 가능해야 합니다.

## 선호 레퍼런스

- `editorial-signal`: 공모전 제안서와 팀 공유용 리포트에 사용합니다. 검정 중심, 표 중심, 근거 중심의 문서 톤을 유지합니다.
- `analyst-workbench`: 보험사 직원용 리포트, score table, decision table, 고객 케이스 비교에 사용합니다.
- 생성된 SVG figure와 결과 리포트는 `.codex-loop/assets/registry.json`에 등록할 수 있어야 합니다.

## 프로젝트별 규칙

- 검정, 흰색, 연한 회색을 기본으로 사용합니다.
- 강조색은 상태 구분이 꼭 필요할 때만 제한적으로 사용합니다.
- 시각화 자료는 `AI pipeline`, `score structure`, `decision flow`, `고객별 판단 예시`에 집중합니다.
- 생활권 밖 주행은 자동 위험으로 표현하지 않습니다.
- 보험료 인상, 벌점, 감시처럼 보이는 문구를 쓰지 않습니다.
- 고객 대상 문구는 안내형으로, 보험사 직원용 리포트는 근거형으로 분리합니다.

## 금지 패턴

- 정보량이 너무 적은 페이지와 몇 개 안 되는 박스
- 장식용 동그라미, pill, 비어 있는 강조 도형
- 도우미처럼 설명하는 말투
- 검토 가치가 없는 스크린샷
- 근거 없는 가짜 수치와 과장된 주장
- 빈약한 내용을 레이아웃으로만 가리려는 마감
