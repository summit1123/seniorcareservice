# 모드 계약: 제안서

이 모드는 제안서 패키지, 공모전 제출물, 검토자 대상 문서, 사업 서사에 사용합니다.

## source of truth

- `docs/submissions/proposal.md`
- `.codex-loop/design/DESIGN.md`
- `.codex-loop/design/reference-packs/` 아래 선택한 reference pack
- `.codex-loop/prd/PRD.md`

## 완료 기준

- Markdown 원고가 채팅 답변이 아니라 실제 검토용 문서처럼 읽힙니다.
- 문제, 해결책, 실현 가능성, 사업 경로, 기대 효과가 모두 명시되어 있습니다.
- 필요한 표, 근거 캡처, 비교, workflow 설명이 source에 존재합니다.
- PDF는 포장 단계일 뿐, 품질이 source 없이 갑자기 생기지 않습니다.

## 실패 조건

- 원고 내용이 빈약한데 레이아웃이 모든 일을 대신합니다.
- 문서가 도우미가 자기 설명을 하는 말투처럼 들립니다.
- 페이지가 성기거나, 박스 위주인데 정보는 적습니다.
- 현재 구현 근거와 미래 확장 계획이 뒤섞여 있습니다.
